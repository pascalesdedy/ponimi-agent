# General Test Case Guidelines

## Coverage Requirements
- Each API endpoint must have at least: 1 happy path, 1 error case, 1 boundary test
- Each UI page must have: load test, interaction test, error state test
- Include at least 1 security-related test case per feature

## Format
- Use descriptive test scenario names
- Steps should be atomic and numbered
- Expected results must include specific values, not vague statements
